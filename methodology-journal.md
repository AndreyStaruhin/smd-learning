## [2026-04-20] — Вопрошание как элемент ядра: двойная рефлексия и Evidence

### Контекст
Исследовательский диалог с переходом в проектировочный. Соавтор предложил изучить два текста из СМД-традиции: "Навигатор вопрошания" (Карастелёв, 2023) и "Практикум интерактивного вопрошания" (Карастелёв, Данилова, 2024). Цель — найти точки усиления методологического ядра.

### Ход обсуждения

1. Изучены оба текста. Навигатор — панорама практик вопрошания (бизнес, образование, философия, техники). Практикум — операциональная система: 50+ техник, формат сессий интерактивного вопрошания, позиционное вопрошание.

2. Выделены три разрыва в текущем ядре:
   - (A) Асимметрия вопрошания: Claude задаёт вопросы, учащийся отвечает, но не учится ставить вопросы сам
   - (B) Типология вопросов не эксплицирована в описании позиций
   - (C) Качество вопросов учащегося не используется как индикатор Evidence

3. Из трёх разрывов решено работать с (A) и (C). Разрыв (B) отложен — требует больше эмпирики.

4. Ключевые концепции из источников, использованные в проектировании:
   - Три адресата вопросов (Данилова-Карастелёв): к себе / к другому / к ноосфере
   - Типология ММК: рефлексивные / на понимание / фокусирующие / об основаниях
   - Вопрошание в ОДИ: от общих вопросов к конкретным через категории и схемы; схема — одновременно ответ и основание для новых вопросов
   - Принцип авторов: вопрошание — не технология, а искусство (согласуется с СМД-принципом мыследеятельности)

5. Для разрыва (A) предложены два варианта переформулировки третьего вопроса двойной рефлексии. Выбран вариант 2 (компактный): один вопрос с осознанной адресацией вместо пассивного "что осталось неясным".

6. Для разрыва (C) спроектирована подсекция в §6: R/T/C-типология вопросов как индикатор уровня освоения, связанный с позициями Claude.

### Принятые решения (уровень D)

**Эмпирическая основа:** наблюдение соавтора о недостаточной активности позиции вопрошания учащегося + теоретический анализ текстов Карастелёва-Даниловой, выявивший разрыв между СМД-принципом "вопрос как продуктивное действие" и текущей практикой ядра.

**Статус решений:** D-гипотеза. Обязательный пересмотр на ближайшей курсовой рефлексии (~через 3 месяца). Критерии оценки:
- Наблюдается ли эволюция вопросов учащегося от R к T/C?
- Помогает ли адресация вопроса (к себе / к другому / к ноосфере) делать рефлексию глубже?
- Не утяжеляет ли новый формат двойную рефлексию?

1. **Двойная рефлексия (§5):** третий вопрос заменён с "что осталось неясным?" на "сформулируй один вопрос — к себе, к предмету, или требующий исследования. Куда он адресован?"

2. **Evidence-индикатор (§6):** добавлена подсекция "Вопрос как индикатор уровня Evidence" с R/T/C-типологией вопросов и связью с позициями Claude.

### Отклонённые варианты

- **Пять вопросов в двойной рефлексии (вариант 1):** отклонено — утяжеляет формат, 5–10 минут на рефлексию не хватит для 5 содержательных ответов. Один вопрос с адресацией достаточен.

- **Отдельная "сессия вопрошания" как паттерн:** отклонено — спекулятивное проектирование без эмпирики. Вопрошание усиливает существующие паттерны, а не создаёт отдельный ритуал.

- **Прямое копирование техник Карастелёва:** отклонено — техники спроектированы для группового формата. Нужна адаптация под пару Claude–учащийся, которая возможна только через эмпирику.

- **Отдельная секция в evidence-ledger для трекинга вопросов:** отложено — reflection-log уже фиксирует вопросы через третий пункт рефлексии. Если через 2–3 месяца эволюция вопросов окажется сильным предиктором, тогда добавится формат.

### Открытые вопросы

- Разрыв (B) — эксплицирование типологии вопросов в описании позиций — требует эмпирики. Стоит вернуться после накопления данных об эволюции вопросов.
- Как адаптировать позиционное вопрошание (Данилова-Карастелёв, из ОДИ) для пары Claude–учащийся? Потенциально мощный инструмент для [Оппонент]-работы.
- Нужен ли "бортовой журнал вопросов" как отдельный артефакт или достаточно reflection-log?
- Техника "Дырявое зеркало" потенциально ценна для модульной мета-рефлексии — исследовать.

### Применённые патчи

- `core/methodology-method.md` §5 — замена третьего вопроса двойной рефлексии
- `core/methodology-method.md` §6 — новая подсекция "Вопрос как индикатор уровня Evidence"
- `core/session-patterns.md` §Этап 6 — зеркальное обновление двойной рефлексии

## [YYYY-MM-DD] — Реструктуризация: выделение course-agnostic ядра

### Контекст

В ходе формирования методологии обнаружена архитектурная ошибка: методологический проект ("Methodology Development") был привязан к конкретному первому курсу (Java + Spring Boot 3). Инструкции проекта содержали программу конкретного курса, позиционную карту Java-мира, специфические модули (G/X/M) и т.п.

Ошибка вскрыта при попытке начать первый методологический диалог в соответствующем Claude Project: Claude справедливо указал, что работа без эмпирики (курс ещё не начался, reflection-log пуст) спекулятивна, и что привязка методологии к одному курсу нарушает изначальную заявку "шаблон для создания шаблонов".

### Ход обсуждения

Проблема осознана через честное наблюдение Claude из методологического проекта. Пользователь подтвердил фундаментальность ошибки. Принято решение провести структурную ревизию **до** начала первого курса — чтобы не накопить технический долг, исправление которого потом будет дороже.

Рассматривались два варианта:
- **A**: подождать с мета-работой до окончания Модуля 0 первого курса (дисциплина "без эмпирики не проектируем")
- **A'**: сделать только структурную ревизию сейчас, без новых спекулятивных конструкций — исправить онтологическую ошибку, не добавляя непроверенного

Выбран вариант A'. Обоснование: сама ревизия не требует эмпирики (это исправление уже вскрытой архитектурной проблемы), но блокирует развитие, если её не сделать.

### Принятые решения (уровень D)

**Выделение методологического ядра `core/` из курсового слоя.**

Созданы документы ядра (все — course-agnostic):
- `core/theoretical-foundations.md` (перенесён из `courses/java-spring-2026/`)
- `core/methodology-method.md` — принципы методологии (новый)
- `core/session-patterns.md` — паттерны сессий (новый)
- `core/artifact-formats.md` — форматы артефактов (новый)
- `core/correction-protocol.md` — 5 контуров + 4 уровня (новый)
- `core/infrastructure-pattern.md` — двухрепо-архитектура как паттерн (новый)
- `core/course-template.md` — шаблон развёртывания курса (новый)

Курсовой слой (`courses/java-spring-2026/`) остаётся как инстанциация, содержит только course-specific материалы — программу, модули, позиции Java-мира, sample code и т.д.

`methodology-journal.md` переносится в корень `smd-learning/`, не внутрь курса.

**Онтологическое обоснование (FPF):**

- `core/*` = U.MethodDescription (описание Method)
- `courses/[курс]/*` = Work (выполнение Method в конкретном Bounded Context)
- Разделение соответствует Strict Distinction (A.7) и принципу Single Source of Truth

**Обоснование (СМД):**

- Методология = работа над средствами (не предметная деятельность)
- Предметные курсы = работа со средствами применительно к предмету
- Методология **объемлет** предметную работу (в терминах ММК), не сливается с ней

### Эмпирическая основа

- Наблюдение из первого методологического диалога: Claude вскрыл архитектурную проблему до того, как она закрепилась в эмпирике
- Указание пользователя на нарушение заявленного свойства методологии ("максимально независимая")
- Теоретическое основание: FPF A.1.1 (Bounded Context) прямо запрещает смешение контекстов разного уровня абстракции

Данные не из проведённого курса (курс ещё не начался), но из проверяемого наблюдения архитектурной ошибки.

### Отклонённые варианты

- **Подождать до Модуля 0**: отклонено. Ошибка уже вскрыта, дальнейшая работа на неправильной основе накапливала бы долг.
- **Оставить методологию привязанной к первому курсу, отделить второй курс позже**: отклонено. Это техническая уступка, которая откладывает проблему. Правильная архитектура сразу делает миграцию опыта между курсами управляемой.
- **Расширить `methodology-project-instructions` без реструктуризации файлов**: отклонено. Онтологическая проблема не решается уровнем инструкций, только на уровне структуры.
