# Методологическое ядро SMD × FPF

> **Этот файл — точка входа.** Прочитай его первым, чтобы понять структуру ядра и найти нужный документ для своей задачи.

## Что это

Учебная методология, синтезирующая системо-мыследеятельностный подход (СМД, Г.П. Щедровицкий) с First Principles Framework (FPF, А. Левенчук). Спроектирована как **шаблон для создания шаблонов** — ядро работает с любым учебным предметом, конкретные курсы являются его инстанциациями.

Цель любого курса по этой методологии — **изменение способа мышления**, не передача содержания.

## Ключевые принципы (ультра-компакт)

Полное описание — в `methodology-method.md`. Здесь — минимум для ориентации:

**Четыре принципа:** онтологичность (цель = сдвиг мышления), многопозиционность (5 позиций Claude), мыследеятельность (три слоя СМД-цикла в каждой сессии), эволюция через эмпирику (правки только на основе данных).

**Пять позиций Claude:** [Наставник] (траектория), [Критик] (проблематизация), [Оппонент] (альтернативные парадигмы), [Симулянт-эксперт] (индустриальные роли), [Методолог] (мета-рефлексия).

**Три уровня Evidence:** R (Reproductive) → T (Transfer) → C (Constructive).

**Четыре уровня коррекции:** A (поведение Claude) → B (структура сессий) → C (программа курса) → D (само ядро).

## Что НЕ входит в ядро

Ядро **course-agnostic**. Следующее определяется при инстанциации под конкретный курс:

- Программа и модули конкретного курса
- Конкретные воплощения [Оппонент] и [Симулянт-эксперт]
- Предметная онтология (onto-map конкретной области)
- Инструменты мыследействия (IDE, среда, язык)
- Сквозной проект

Всё это живёт в `courses/[название-курса]/`, не в `core/`.

## Версионирование

Ядро эволюционирует через `methodology-journal.md`. Каждая правка уровня D фиксируется с эмпирической основой, отклонёнными вариантами и критериями пересмотра. Коммиты — по конвенции из `infrastructure-pattern.md` §8.
