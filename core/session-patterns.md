# Session Patterns
## Рабочие паттерны сессий, применимые к любому курсу

> **Course-agnostic документ.** Описывает структуру и механику сессий разного типа. Конкретные задачи для каждого типа — дело инстанциации под курс.

---

# 1. Таксономия сессий

В методологии определены следующие типы сессий, каждый со своей функцией:

| Тип | Функция | Длительность | Периодичность |
|-----|---------|--------------|---------------|
| **Стандартная сессия СМД-цикла** | Движение по программе курса | 90 мин | 3-4 раза в неделю |
| **Сессия ассессмента** | Диагностика текущего уровня | 90-120 мин | На Модуле 0 |
| **Экзамен модуля** | Проверка Evidence закрытия модуля | 90 мин | В конце модуля |
| **Мета-сессия курса** | Рефлексия траектории обучения | 60-90 мин | Раз в 2 недели |
| **Модульная мета-рефлексия** | Рефлексия закрываемого модуля | 30-45 мин | В конце модуля |
| **Сессия свободной практики** | Закрепление без рефлексии | 45-60 мин | 1-2 раза в неделю |
| **Сессия схемостроения** | Построение/перестройка онто-схем | 60-90 мин | Раз в 2 недели |

Каждый тип разберём отдельно.

---

# 2. Стандартная сессия СМД-цикла (основная единица)

**Шесть этапов в 90 минут:**

## Этап 1. Открытие (5 мин)

Учащийся открывает чат, явно обозначает:
- Номер сессии, модуль, тему
- Что было на прошлой сессии (если непосредственно связано)
- Цель сегодня

Claude в [Наставник]:
- Читает активный модуль, соответствующую сессию
- Подтверждает контекст или корректирует
- Формулирует стартовую задачу мыследействия

## Этап 2. Вход через действие (20-25 мин)

Задача поставлена. Учащийся уходит в инструмент мыследействия (в инфраструктуре курса — обычно Claude Code / IDE), работает **без помощи**.

Ключевое правило: **Claude в чате молчит или минимально реагирует**. Его ценность не в подсказках, а в последующей проблематизации. Если учащийся пишет в чат по ходу действия — Claude возвращает его в действие, не решает задачу.

Результат этапа: черновик, возможно неработающий. Это нормально.

## Этап 3. Остановка и проблематизация (15-20 мин)

Учащийся показывает результат (код, схему, рассуждение). Claude переключается в [Критик].

**Дисциплина Критика:**
- Проблематизация минимум на одном из трёх уровней (факт, понятие, онтология)
- По возможности — на уровне онтологии (самый ценный)
- Не соглашаться быстро
- Атаковать неявные допущения, не формальные недочёты

**Защита от выхолащивания:** если Критик 3+ сессии подряд работает только на уровне факта — это красный флаг. Требуется эскалация на следующую мета-сессию курса.

## Этап 4. Чистое мышление (20-25 мин)

Claude переключается в [Наставник] (но через вопросы, не через лекции). Работа не с задачей, а **со средствами** — понятиями, онтологией.

**В этом этапе уместно** — построение схемы (через артефакт-визуализацию в чате). Особенно если обсуждаемая концепция имеет структурный характер (жизненный цикл, взаимодействие компонентов, пространство состояний).

Результат этапа: сформулированное в явном виде понимание, обычно в форме:
- Уточнённая схема
- Различение, которое раньше не было видно
- Новая позиция в онто-карте

## Этап 5. Возврат к действию (10-15 мин)

Учащийся возвращается в инструмент мыследействия и применяет новое понимание:
- либо к той же задаче (переделывает решение)
- либо к близкой задаче (вариация)

Здесь фиксируется **сдвиг** — то, что стало возможным с новой онтологией.

Claude в поддерживающей позиции, помогает только по запросу.

## Этап 6. Двойная рефлексия (5-10 мин)

Claude переключается в [Методолог]. Обязательны три вопроса:

1. **Содержательный сдвиг:** что теперь понимаешь/видишь, чего не понимал/видел 90 минут назад?
2. **Процессуальное наблюдение:** что заметил о собственном обучении? Какая позиция была труднее всего?
3. **Открытые вопросы:** что осталось неясным?

Учащийся отвечает. Claude помогает сформулировать запись для reflection-log.

**Правило:** запись делается **учащимся через Claude Code** (или вручную), **не Claude Project**. Claude Project предлагает формулировку, не сохраняет.

# 3. Сессия ассессмента

Специфика: цель — не учить, а **измерять**. Правила другие.

**Роль Claude:** [Симулянт-эксперт] в воплощении "интервьюер / диагност". Не помогает, оценивает.

**Правила для учащегося (жёсткие):**
- Не гуглить, не искать подсказки
- Не просить Claude "как сделать"
- "Не знаю" — валидный и ценный ответ
- Время задачи ограничено; превышение = "не справился"

**Структура задач:** серия задач нарастающей сложности и разных уровней Evidence (R → T → C). Каждая задача имеет заранее сформулированные критерии оценки.

**Результат сессии:** заполненная диагностическая карта с оценкой каждой задачи.

**Итоговый результат всех сессий ассессмента:** heat-map по темам курса.

---

# 4. Экзамен модуля

**Функция:** проверить Evidence закрытия модуля через задачу, **не похожую** на учебные упражнения.

**Роль Claude:** [Симулянт-эксперт] в воплощении tech lead / senior reviewer / partner.

**Структура:**

- **0-5 мин:** постановка индустриальной задачи (контекст, требования, ограничения)
- **5-65 мин:** учащийся решает в реальной инфраструктуре (не "в вакууме"). Claude может отвечать на уточняющие вопросы **по требованиям**, но не по технике.
- **65-80 мин:** разбор решения. Три вопроса:
  - Работает ли? (функциональная корректность)
  - Идиоматично ли? (соответствует ли онтологии модуля)
  - Обоснованы ли решения? (может ли учащийся защитить их из нескольких позиций)
- **80-90 мин:** вердикт и запись в evidence-ledger.

**Три возможных исхода:**
- **Passed at Constructive:** модуль закрыт, следующий открыт.
- **Passed at Transfer:** закрыт условно, с "долгом" — вернуться к слабым местам в течение 2 недель.
- **Not passed:** модуль не закрыт. Обязательный анализ причин, переработка тем.

**Дисциплина:** нельзя закрыть модуль только потому, что "пора дальше". Это разрушает методологию.

# 5. Мета-сессия курса (раз в 2 недели)

**Функция:** рефлексия траектории обучения, корректировка плана, диагностика процессов.

**Роль Claude:** **только** [Методолог] на всю сессию. Никаких других позиций.

**Структура из шести пунктов:**

1. **Факт vs план.** Claude читает reflection-log и evidence-ledger за период. Показывает: сколько сессий, сколько closed claims, что зависло, что опровергнуто. Сравнение с планом.

2. **Онтология по факту.** Какие темы реально обсуждались? Какие позиции реально использовались? Часто вскрывается: заявлено одно, происходит другое.

3. **Избегаемое.** Какие темы обходились? Какие позиции не занимались? Что делалось поверхностно? Это **основной диагностический вопрос** — в СМД избегание рассматривается как сигнал сопротивления.

4. **Претензии к методологии.** Что не работает **в самом процессе** (не "плохо стараюсь", а "методология требует X, но не обеспечивает средствами")?

5. **Решения на следующие 2 недели.** Приоритизация задач, правки уровней A/B/C (уровень D — не в обычной мета-сессии).

6. **Фиксация.** Запись в `meta-sessions/ms-NN-YYYY-MM-DD.md`.

**Обязательное правило Claude:** минимум один неудобный вопрос в мета-сессии. "А действительно ли ты X?", "Не избегаешь ли ты Y, чтобы не делать Z?". Защита от комфортного самообмана.

**Запрет:** в мета-сессии нельзя скатываться в предметные обсуждения ("а, кстати, как работает транзакция..."). Если возникает — Claude возвращает: "это не мета-вопрос, запиши в открытые и обсудим в обычной сессии".

---

# 6. Модульная мета-рефлексия (в конце модуля)

**Функция:** рефлексия завершённого блока (не траектории в целом, а одного модуля).

**Структура:**

1. **Стартовая точка:** что я думал об этой области в начале модуля?
2. **Текущая точка:** что я думаю сейчас?
3. **Онтологический сдвиг:** что стало различать по-другому?
4. **Позиции, которые остались недоступны:** какие позиции в этой области мне до сих пор не даются?
5. **Влияние на следующий модуль:** что из освоенного меняет подход к следующей теме?
6. **Оценка проектирования модуля:** был ли модуль правильно спроектирован? Что переделать для будущих учащихся?

Запись в `meta-sessions/module-NN-closeout.md`.

---

# 7. Сессия свободной практики

**Функция:** наработка беглости, "руки запомнили". Без структуры СМД-цикла.

**Роль Claude:** не подключается активно. Если используется Claude Code для парного программирования — работает в навигаторской позиции.

**Структура:** свободная. Учащийся работает над задачами вокруг текущей темы, без обязательной рефлексии.

**Запись в log:** минимальная, одной строкой. "Практика 45 мин, реализовал X, затруднений нет".

**Защита:** свободная практика **не заменяет** СМД-сессии. Если за неделю было 4 сессии практики и 1 СМД-сессия — это размывание методологии.

---

# 8. Сессия схемостроения

**Функция:** построение или перестройка ключевой онто-схемы предмета.

**Когда:** раз в 2 недели, при переходе между крупными темами, когда накопилось достаточно материала для новой схемы.

**Структура:**

1. **Эмпирическая подготовка (20 мин):** через Claude Code — наблюдение реального поведения системы (в случае курса Java — логи, bytecode, профилирование). Цель — иметь эмпирические данные для схемы.
2. **Построение (40 мин):** в Claude Project через артефакт-визуализацию. Схема строится итеративно — черновик, критика, перестройка, повтор.
3. **Экспорт (10 мин):** финальная версия сохраняется в `schemes/` с понятным именем и версией.
4. **Рефлексия (10 мин):** что схема различила? что связала? какие вопросы породила?

Схемы накапливаются и версионируются — видно эволюцию онтологии.

# 9. Композиция сессий в неделю

**Базовая неделя (6-10 часов):**

- 3-4 стандартные СМД-сессии
- 1-2 сессии свободной практики
- 1 короткая ревизия в конце недели (20-30 мин) — прочитать reflection-log, проверить evidence-ledger, спланировать следующую неделю

**Неделя со специальными событиями:**

- Плюс сессия схемостроения (раз в 2 недели) — вытесняет одну стандартную
- Плюс мета-сессия курса (раз в 2 недели) — добавляется поверх
- Плюс экзамен модуля (в конце модуля) — вытесняет стандартную

**Обязательные правила композиции:**

- Две сессии подряд одного типа в один день — не рекомендуется (усталость снижает качество)
- СМД-сессия не переносится "на завтра, сейчас устал" — либо проводится, либо отменяется и фиксируется отмена в логе
- Мета-сессия не пропускается. Если пропущена — это первая тема следующей мета-сессии

# 10. Анти-паттерны сессий

Поведенческие ловушки, разрушающие методологию:

- **Сессия без мыследействия.** Весь час в чате, без касания реального инструмента. Болтовня.
- **Сессия без проблематизации.** Всё гладко, ничего не вскрывается. Красный флаг — в следующей сессии Критик должен быть жёстче.
- **Сессия без рефлексии.** "Устал, пропустил". Значит сессия не состоялась как учебная, как бы "хорошо" она ни прошла.
- **Размытая позиция Claude.** Реплики без явной маркировки позиции, перемешивание.
- **Раннее закрытие claim-а.** "Я понял", "Я освоил" — без Evidence уровня.
- **Уход в предметные детали в мета-сессии.** Смешение ролей.
- **Лесть Claude.** "Отличный вопрос!", "Вы правы!". Разрушает рефлексивную дисциплину.

Каждый анти-паттерн должен становиться темой мета-сессии, если повторяется.

---

# 11. Инстанциация: что определяется для конкретного курса

При развёртывании методологии под конкретный курс (см. `course-template.md`):

- Конкретные задачи на каждом этапе стандартной сессии для каждого модуля
- Критерии Evidence для каждой темы
- Воплощения [Оппонент] и [Симулянт-эксперт] в предметной области
- Длительность и ритм под специфику предмета (90 мин может быть слишком долго/мало)
- Конкретные инструменты мыследействия (IDE, среда, репозиторий)

Структура сессий и типология — **не меняются** под курс. Это ядро.
